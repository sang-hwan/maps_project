---

### 📌 장소 데이터 수집 및 정리 세부 실행 계획 (2\~3주)

---

**1. 데이터 수집 대상과 범위 명확화**

* 대상 지역을 '대전광역시'로 한정하고, 수집 대상 업종을 '카페, 바, 동전 노래방'으로 명확히 설정합니다.
* 데이터 수집을 위한 지역의 행정구역(구, 동) 리스트를 정리하여 데이터 수집의 범위를 명확하게 구획합니다.

**2. API를 활용한 데이터 본격 수집**

* 1단계에서 구축한 데이터 수집 프로그램을 이용하여 Google Maps, 네이버 지도, 카카오맵 API로부터 장소 데이터를 본격적으로 수집합니다.
* 데이터는 지역(구, 동)과 카테고리(카페, 바, 동전 노래방) 기준으로 각각 나누어 수집합니다. 데이터 누락을 최소화하기 위해 각 플랫폼의 데이터를 병행하여 수집합니다.

**3. 중복 데이터 처리 로직 적용**

* 서로 다른 API에서 같은 장소 정보가 중복 수집되는 경우가 많으므로, 장소명과 주소 등을 기준으로 중복 여부를 판단하여 중복 데이터를 필터링하는 로직을 적용합니다.
* 각 장소의 대표 데이터를 선정하는 기준(정확성, 최신성 등)을 마련하여 중복된 정보 중 가장 신뢰할 수 있는 데이터를 선택합니다.

**4. 운영 상태(영업 중/폐업) 확인 자동화**

* 수집된 장소 데이터를 대상으로 각 플랫폼에서 제공하는 '운영 상태' 관련 정보를 활용하여 장소가 현재 운영 중인지 폐업했는지를 자동으로 판단합니다.
* 자동으로 확인이 어려운 데이터의 경우, 추가로 웹사이트(예: 네이버 블로그, 공식 SNS, 리뷰 사이트 등)를 크롤링하여 운영 여부를 크로스체크하는 기능을 구축합니다.

**5. 데이터 신뢰성 및 품질 검증**

* 자동으로 수집된 데이터를 대상으로 무작위 샘플링을 통해 일정 비율의 데이터 정확성을 수동으로 검증합니다.
* 특히 운영 상태가 불확실한 데이터는 수작업으로 추가 검증하고, 신뢰성에 문제가 발견될 경우 데이터 수집 및 필터링 로직을 개선합니다.

**6. 최종 리스트 구성**

* 검증된 장소 데이터 중 '운영 중'인 장소만을 별도의 최종 데이터 리스트로 추출하여 정리합니다.
* 정리된 리스트에는 최소한 '장소명', '주소', '카테고리', '운영 상태'가 포함되도록 구성하며, 향후 사용을 위해 데이터를 표준화된 형태로 정돈합니다.

**7. 데이터 보완 작업**

* 수집 결과에서 데이터가 누락되거나 불충분하다고 판단되는 지역 또는 카테고리에 대해 재수집하거나 추가적으로 웹 크롤링 등 별도의 방법을 이용하여 보완합니다.
* 보완 작업은 특히 소규모 가게나 최근에 개업한 가게처럼 API로 쉽게 찾을 수 없는 장소를 대상으로 이뤄집니다.

**8. 최종 결과 리뷰 및 보고서 작성**

* 최종적으로 정리된 장소 데이터를 한 번 더 리뷰하고, 전체 수집 결과를 간략히 분석하여 데이터 수집 프로세스의 효율성과 문제점 등을 기록합니다.
* 이 리뷰 보고서는 향후 데이터베이스 구축 단계(3단계)에서 데이터 입력 시 주의해야 할 점이나 추가 관리가 필요한 부분에 대한 가이드라인으로 사용됩니다.

이러한 세부 계획을 통해 체계적이고 신뢰성 높은 장소 데이터 수집 및 정리를 마칠 수 있습니다.
